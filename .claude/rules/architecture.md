# Architecture Rules — pipeline + facade (local CLI)

> Generated by the prd-to-architecture skill. **Never edit by hand** — regenerate through the skill. These rules are enforced by module-audit before every push.

Equivalent of the Modular-Monolith ruleset, adapted to this tool's orchestrated-pipeline + facade architecture.

## Structure

- **The CLI is a thin shell.** `export.py` may only: parse flags, build `RunContext`, call `run_export(config)`, and map `WaeError`/`KeyboardInterrupt` to exit codes. No business logic, no adb, no file I/O beyond reading `--key-file`.
- **All orchestration lives behind the facade.** `run_export(config) -> Path` is the single entry to the pipeline. Tests and any programmatic caller use it, not `export.py`.
- **No facades-in-front-of-facades.** One package-level facade only; the internal orchestrator is a coordinator, not a second facade.

## Module isolation

- **No cross-module imports between stages.** `env_check`, `keyutil`, `pull`, `decrypt_export`, `contacts`, `package` never import one another. They communicate only through the orchestrator, which passes the immutable `RunContext` and each stage's returned values.
- **`RunContext` is frozen and carries non-secret config only.** No mutation; no shared mutable state.
- **One responsibility per module.** If a module's responsibility needs the word "and", split it.

## The key (hard rules)

- **The key lives only in `keyutil` (produces) and `decrypt_export` (consumes).** It is passed as a separate argument — never placed in `RunContext`.
- **The key is never logged, printed, written to disk, or included in error messages.** Any logged adb/exporter command must replace the key with `***`.
- **`SecretRedactionFilter` is mandatory** on the logger as a backstop: any 64-hex substring in a log record is redacted.

## External boundaries (adapters)

- **`adb` is invoked only from `pull`.** No other module shells out to a device.
- **The device is read-only.** No `adb` command may write to, push to, or otherwise modify the phone. `adb shell` is allowed only for read-only probes (`ls`, `stat`, `du`, `version`).
- **All decryption/parsing goes through `whatsapp-chat-exporter`.** No custom cryptography anywhere.
- **adb retry policy:** 3 attempts, linear backoff, transport-level errors only. Never retry logical errors (file-not-found).

## Network & config

- **No network, anywhere.** No `http`, `socket`, `requests`, `urllib`, or equivalent imports in `wae/` or `export.py`. No telemetry.
- **No env-based secrets.** The key comes only from the interactive prompt or `--key-file`, passed explicitly. No `os.environ` reads for secrets or config that belongs on the CLI.

## Errors & lifecycle

- **Raise typed errors.** Stages raise `WaeError` subclasses (`EnvError`/`DeviceError`→1, `InvalidKey`→2, `NoBackupError`→3, `DecryptionError`→4, `PackagingError`→5). No scattered `sys.exit`; only the shell exits.
- **Decryption must be verified.** Success requires exporter exit 0 **and** expected output present. Never leave partial output on failure.
- **Guaranteed teardown.** Temp workdir wipe and key wipe run via `try/finally` (or context manager) on every exit path, including `KeyboardInterrupt` (exit 130). `--keep-temp` is the only exception to temp wipe.

## Secrets in Git

- **`.gitignore` must exclude, from the first commit:** `key`, `*.key`, `key.txt`, `*.crypt12`, `*.crypt14`, `*.crypt15`, `msgstore.db`, `wa.db`, `*.vcf`, `/output/`, `/.tmp/`, `*.zip`, `.env`.
- **Hard stop on push** if any of these would be committed.

## Deferred edge cases

- Anything marked Deferred in SPEC §11 (legacy crypt12/crypt14, cloned-app installs, oversized single chat, timezone customization) must **fail clearly** with the correct exit code and an actionable message — never degrade silently.
