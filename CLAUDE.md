## Core rules (summary â€” full detail in `.claude/rules/architecture.md`)

1. **CLI stays thin.** All orchestration lives behind the `run_export` facade â€” never in `export.py`.
2. **No cross-module imports.** Stages talk only via the orchestrator (immutable `RunContext` + returned values).
3. **`adb` only in `pull`.** No other module shells out to a device.
4. **The key only in `keyutil` and `decrypt_export`.** Never in `RunContext`, never logged, never printed, never written.
5. **No custom crypto.** All decryption/parsing via `whatsapp-chat-exporter`.
6. **No network, anywhere.** No http/socket imports; no telemetry.
7. **Device is read-only.** No `adb` command may write to or modify the phone.
8. **Typed errors â†’ exit codes.** Raise `WaeError` subclasses; the shell maps them. No scattered `sys.exit`.
9. **Guaranteed teardown.** Temp wipe + key wipe via `try/finally` on every exit path (incl. Ctrl-C â†’ 130).
10. **Secrets never committed.** `.gitignore` excludes key, `*.crypt15`, `msgstore.db`, `wa.db`, `*.vcf`, `/output/`, `/.tmp/`, `*.zip`.
